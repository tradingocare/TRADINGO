AUTH FLOW DIAGRAM
=================

1. REGISTRATION
───────────────

  Client                    API                         DB
   │                         │                          │
   │  POST /auth/register    │                          │
   │  {email,password,name}  │                          │
   │────────────────────────>│                          │
   │                         │  Check if email exists   │
   │                         │─────────────────────────>│
   │                         │  User not found          │
   │                         │<─────────────────────────│
   │                         │                          │
   │                         │  Hash password (bcrypt)  │
   │                         │  Create user             │
   │                         │─────────────────────────>│
   │                         │                          │
   │                         │  Generate JWT Access     │
   │                         │  Token (15m) + Refresh   │
   │                         │  Token (UUID, 7d)        │
   │                         │                          │
   │                         │  Store Refresh Token     │
   │                         │  in Session table        │
   │                         │─────────────────────────>│
   │                         │                          │
   │  {user, accessToken,    │                          │
   │   refreshToken}         │                          │
   │<────────────────────────│                          │

2. LOGIN
────────

  Client                    API                         DB
   │                         │                          │
   │  POST /auth/login       │                          │
   │  {email,password}       │                          │
   │────────────────────────>│                          │
   │                         │  Find user by email      │
   │                         │─────────────────────────>│
   │                         │  Return user + hash      │
   │                         │<─────────────────────────│
   │                         │                          │
   │                         │  bcrypt.compare()        │
   │                         │  └── if invalid → 401    │
   │                         │                          │
   │                         │  Generate JWT:           │
   │                         │  {sub, email, role,      │
   │                         │   permissions, iat, exp} │
   │                         │                          │
   │                         │  Generate Refresh Token  │
   │                         │  Store in Session table  │
   │                         │─────────────────────────>│
   │                         │                          │
   │  {user, accessToken,    │                          │
   │   refreshToken}         │                          │
   │<────────────────────────│                          │

3. AUTHENTICATED REQUEST (JWT Guard)
─────────────────────────────────────

  Client                    API                         DB
   │                         │                          │
   │  GET /api/resource      │                          │
   │  Authorization:         │                          │
   │  Bearer <accessToken>   │                          │
   │────────────────────────>│                          │
   │                         │                          │
   │  ┌─ JwtAuthGuard ──────┤                          │
   │  │  Extract JWT from   │                          │
   │  │  Authorization      │                          │
   │  │  header             │                          │
   │  │                     │                          │
   │  │  Verify signature   │                          │
   │  │  + expiry           │                          │
   │  │                     │                          │
   │  │  Validate payload   │                          │
   │  │  against DB user    │                          │
   │  │─────────────────────│──────────────────────────>│
   │  │  User found+active  │                          │
   │  │<────────────────────│──────────────────────────│
   │  │                     │                          │
   │  │  Attach payload to  │                          │
   │  │  req.user           │                          │
   │  └─────────────────────┤                          │
   │                         │                          │
   │  ┌─ RolesGuard ────────┤                          │
   │  │  Check @Roles()     │                          │
   │  │  against req.user   │                          │
   │  │  .role              │                          │
   │  │  └── if fail → 403  │                          │
   │  └─────────────────────┤                          │
   │                         │                          │
   │  ┌─ PermissionsGuard ──┤                          │
   │  │  Check @Permissions()│                          │
   │  │  against req.user   │                          │
   │  │  .permissions[]     │                          │
   │  │  (OR logic)         │                          │
   │  │                     │                          │
   │  │  SUPER_ADMIN        │                          │
   │  │  bypasses all       │                          │
   │  │  └── if fail → 403  │                          │
   │  └─────────────────────┤                          │
   │                         │                          │
   │  ┌─ Controller ────────┤                          │
   │  │  Actual handler     │                          │
   │  │  executes           │                          │
   │  └─────────────────────┤                          │
   │                         │                          │
   │  Response               │                          │
   │<────────────────────────│                          │

4. TOKEN REFRESH (Rotation)
────────────────────────────

  Client                    API                         DB
   │                         │                          │
   │  POST /auth/refresh     │                          │
   │  {refreshToken}         │                          │
   │────────────────────────>│                          │
   │                         │                          │
   │                         │  Find session by         │
   │                         │  refreshToken            │
   │                         │─────────────────────────>│
   │                         │                          │
   │                         │  if not found → 401      │
   │                         │  if expired → 401        │
   │                         │                          │
   │                         │  DELETE old session      │
   │                         │  (rotation: old token    │
   │                         │   can't be reused)       │
   │                         │─────────────────────────>│
   │                         │                          │
   │                         │  Generate NEW access     │
   │                         │  token + NEW refresh     │
   │                         │  token                   │
   │                         │                          │
   │                         │  Store new refresh token │
   │                         │─────────────────────────>│
   │                         │                          │
   │  {accessToken,          │                          │
   │   refreshToken}         │                          │
   │<────────────────────────│                          │

5. TOKEN STRUCTURE
───────────────────

  Access Token (JWT):
  ┌─────────────────────────────────────┐
  │ Header: {alg: "HS256", typ: "JWT"} │
  │ Payload: {                          │
  │   sub: "uuid",           ← userId  │
  │   email: "a@b.com",                │
  │   role: "ADMIN",                   │
  │   permissions: ["read:orders"],    │
  │   iat: 1700000000,                 │
  │   exp: 1700000900                  │
  │ }                                   │
  │ Sign: HMAC-SHA256(JWT_SECRET)       │
  └─────────────────────────────────────┘
  TTL: 15 minutes

  Refresh Token:
  ┌─────────────────────────────────────┐
  │ UUID v4 (opaque, stored in DB)     │
  │ Bound to user via Session table    │
  │ Has explicit expiresAt column      │
  └─────────────────────────────────────┘
  TTL: 7 days
